Packages

Before using these packages, make sure you have node.js installed on your platform. You can find that here.
Now, these are some packages that are commonly used when creating a new project with a node server. This uses the terminal commands with gitbash.

  • npm: this install allows you to download any package or any dependencies within a package
  • express: within node js, this helps to provide the features that work best when creating an app. It's a library that makes deploying a webserver easier.
    to install type: npm i express
  • ejs: this allows you to be able to create html template with the least amount of code. Places plain JavaScript into the html markup. (hence embedded JS templating)
    to install type: npm i ejs
  • nodemon: this helps to automatically restart/refresh the application anytime a new change is made. Will also help to redownload packages if they do not show up.
    to install type: npm i nodemon

Issues that can be checked

There are many instances that can happen when the packages do not install correctly, here are some ways to check before continuing on with the project. et. pushing and deploying.

  1. Did the packages pop up in the file explorer after downloading a package?
    - if not try typing npm init -y, this will create a node.js project and also create the packages that may be missing
  2. Did you project build successfully but then fail deployment because of a missing package?
    - If you downloaded the packages (for example express) check the dependencies in the package-lock-json folder. The picture below is what that should look like, along with whatever version you are using.
  • "dependencies": {
    "ejs": "^3.1.8",
    "express": "^4.18.2",
    "nodemon": "^2.0.20"
    }
