Packages

Here are some packages that are commonly used when creating a new project

  • npm: this install allows you to download any package or any dependencies within a package
  • express: within node, this helps to provide the features that work best when creating an app
  • ejs: this allows you to be able to create html template with the lest amount of code. Helps to place code into the client side.
  • nodemon: this helps to automatically restart/refresh the application anytime a new change is made.

Issues that can be checked

There are many instances that can happen when the packages do not install correctly, here are some ways to check before continuing on with the project. et. pushing and deploying.

  • Did the packages pop up in the file explorer after downloading a package?
  • When downloading ejs, the json packages should appear. If this is not the case, try typing npm init
  • If packages downloaded and you do not want the deployment failed, check the dependencies in either of the json packages.
